# 置信度功能 · 核心说明（给队友）

**做的事**：让 AUR 的每个模态不只输出情绪，还给每个情绪一个 0~1 的置信度。
**前提**：输入不变、模型不变，只改提示词 + 输出解析（方案 A：模型自报分，1 次生成）。

---

## 1. 使用的模型
- **只有一个模型：AffectGPT**（权重 `checkpoint_000035_loss_0.716.pth`，LLaMA + LoRA，约 7.66B 参数）。
- 4 个模态（audio / face / text / multi）**都是这同一个 AffectGPT 跑的**，不是 4 个模型。
- 本次测试**未用** Qwen2.5 / Qwen3（它们是抽标签 / EoP 用的）。

## 2. 每个模态的提示词
**指令部分 4 个模态完全相同**（`AUR_CONF_PROMPT`，原文）：
> Please identify all possible emotional states of the character, and assign each one a confidence score between 0.0 and 1.0 (higher means more certain). Output ONLY a comma-separated list in the exact format 'emotion: score', and nothing else. Example: anxious: 0.8, tense: 0.5, calm: 0.2

**仅"输入描述"段不同**（各喂不同模态，指令都接在后面）：
| 模态 | 输入段 |
|---|---|
| audio | `The audio content is: <Audio><AudioHere></Audio>.` |
| face | `...extract faces...: <Video><FaceHere></Video>.` |
| text | `The subtitle is: <Subtitle>{字幕}</Subtitle>.` |
| multi | `<Multi>…<Audio>…<Face>…<Subtitle>…<Caption>…`（带 caption） |

> 注：`<AudioHere>`/`<FaceHere>` 等是占位符，运行时被替换成该模态编码后的特征向量（不是文字）。

## 4. 运行的脚本
- **脚本**：`inference_emosync.py`（加了置信度的版本）。
- **命令**（产出 `..._conftest.npz`）：
```
CUDA_VISIBLE_DEVICES=0 /opt/data/wlcc/Software/Anaconda3/envs/affectgpt/bin/python -u inference_emosync.py \
  --cfg-path=train_configs/mercaptionplus_outputhybird_bestsetup_bestfusion_face_lz.yaml \
  --dataset MER2026OV --train_debug --options 'inference.test_epoch=35' \
  --max_samples 3 --save_tag _conftest
```
