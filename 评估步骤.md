# 1. MER2026 Track 2 最终成果评估步骤

## 1.1 评估目标

MER2026 Track 2，即 MER-FG，评估的是开放词汇细粒度情绪识别结果。模型可以为每个样本输出任意数量、任意类别的情绪标签。

最终成果不是直接按模型生成的原始解释文本评分，而是按以下流程评分：

```text
模型输出描述 -> 抽取 open-set 情绪标签 -> emotion wheel 归一映射 -> 计算 precision/recall -> 计算 5 个 emotion wheels 的平均 F1
```

最终分数越高越好。

## 1.2 第一步：生成模型推理结果

模型首先需要对每个测试样本生成情绪相关的推理文本或描述。

在当前项目中，AffectGPT 推理脚本会为每个样本生成 `reason`，并保存为 `.npz` 文件。

典型保存路径为：

```text
output/results-mer2026ov/<model_name>/*.npz
```

其中 `.npz` 文件中通常包含：

```text
name2reason
```

`name2reason` 表示：

```text
样本名 -> 模型生成的情绪推理/描述
```

## 1.3 第二步：将推理文本转成 open-set 情绪标签

由于最终评估的是情绪标签，而不是完整自然语言解释，因此需要把模型生成的 `reason` 转换成开放词汇情绪标签。

当前项目中可以运行：

```bash
python ovlabel_extraction.py
```

该脚本会调用 Qwen2.5，将每条 reason 转换成类似如下格式：

```text
[happy, anxious, disappointed]
```

转换后的结果会保存为：

```text
xxx-openset.npz
```

其中通常包含：

```text
filenames
fileitems
```

含义为：

```text
filenames: 样本名列表
fileitems: 每个样本对应的 open-set 情绪标签
```

## 1.4 第三步：读取真实标签和预测标签

评估脚本会读取真实标签，即 ground truth。

在当前项目中，路径由 `config.py` 中的配置决定：

```python
config.PATH_TO_LABEL['MER2026OV']
```

当前配置指向：

```text
track2_test.csv
```

该文件中关键字段通常包括：

```text
name
openset
```

其中：

```text
name: 样本名
openset: 真实 open-set 情绪标签
```

需要注意：正式比赛的最终测试集标签通常不会公开。参赛者只能在本地验证集上自测，最终官方会在私有标签上计算排行榜分数。

## 1.5 第四步：加载 emotion wheel 映射文件

Track 2 使用 Emotion Wheel-based 指标来处理开放词汇标签、同义词和标签变体。

当前项目中的映射文件为：

```text
emotion_wheel/wheel_mapping.npz
```

该文件由 `config.py` 中的变量指定：

```python
OUTSIDE_WHEEL_MAPPING = emotion_wheel/wheel_mapping.npz
```

映射文件中主要包含三类映射：

```text
format_mapping
raw_mapping
wheel_map_whole
```

它们分别用于：

```text
format_mapping: 词形或格式归一
raw_mapping: 同义词映射
wheel_map_whole: emotion wheel 层级映射
```

## 1.6 第五步：对标签进行归一化处理

评估时，真实标签和预测标签都会经过相同的归一化流程。

主要处理步骤为：

```text
1. 转成小写
2. 去除前后空格
3. 词形归一
4. 同义词映射
5. 映射到 emotion wheel 中的对应层级
6. 删除无法映射的无效标签
7. 使用 set 去重
```

例如：

```text
angered -> anger
joyful -> happy
terrified -> fear
```

如果某个预测词不在 emotion wheel 映射体系中，它不会参与最终计分。

## 1.7 第六步：逐样本计算 precision 和 recall

对于每个样本，评估脚本会比较归一化后的真实标签集合和预测标签集合。

记：

```text
GT = 真实标签集合
Pred = 预测标签集合
```

则每个样本的 precision 和 recall 为：

```text
precision = |GT ∩ Pred| / |Pred|
recall    = |GT ∩ Pred| / |GT|
```

如果预测标签为空，则该样本的 precision 和 recall 都记为 0。

这一点非常重要：

```text
预测标签太多会降低 precision
预测标签太少会降低 recall
```

因此，最终模型不能简单地输出越多情绪词越好，而要控制每个样本的标签数量。

## 1.8 第七步：在 5 个 emotion wheels 上分别计算 F1

Track 2 默认使用 5 个 emotion wheels 进行评估。

当前项目中默认的 level1 评估项为：

```text
case3_wheel1_level1
case3_wheel2_level1
case3_wheel3_level1
case3_wheel4_level1
case3_wheel5_level1
```

每个 wheel 都会得到一个 precision、recall 和 F1。

F1 的计算公式为：

```text
F1 = 2 * precision * recall / (precision + recall)
```

如果 precision 和 recall 同时为 0，则 F1 记为 0。

## 1.9 第八步：计算最终分数

最终分数是 5 个 emotion wheels 的 F1 平均值。

公式为：

```text
final_score = mean(F1_wheel1, F1_wheel2, F1_wheel3, F1_wheel4, F1_wheel5)
```

当前项目中的 `evaluation.py` 会将最终 F1 转成百分制输出：

```text
score = final_score * 100
```

因此，如果输出为：

```text
72.35
```

表示最终平均 F1 为：

```text
0.7235
```

## 1.10 第九步：本地评估命令

在已经生成 open-set 标签或 reason 结果后，可以运行：

```bash
python evaluation.py
```

该脚本会自动遍历：

```text
output/results-mer2026ov/*/*.npz
```

如果对应的 `-openset.npz` 不存在，脚本会先调用 Qwen2.5 抽取 open-set 标签。

随后脚本会计算 EW-based 分数，并打印类似结果：

```text
<model_name> & 72.35 \\
```

其中 `72.35` 就是百分制形式的最终平均 F1。

## 1.11 第十步：优化最终成果时最应该关注的点

由于最终分数由 emotion wheel 映射后的平均 F1 决定，因此优化重点包括：

```text
1. 提高情绪标签召回率，避免漏掉关键情绪
2. 控制每个样本输出标签数量，避免 precision 明显下降
3. 使用 emotion wheel 友好的标准情绪词
4. 删除不明确、不可映射、非情绪类词汇
5. 对同义词和近义词进行统一
6. 在 Human-OV 验证集上搜索最佳标签数量和融合阈值
7. 对多个模型、多个 epoch、多个 prompt 的结果做 ensemble
```

## 1.12 总结

MER2026 Track 2 的最终评估并不是简单比较原始文本，而是比较经过 emotion wheel 归一后的开放词汇情绪标签。

完整评估链路为：

```text
1. 模型生成 reason
2. reason 转 open-set 情绪标签
3. 读取真实标签和预测标签
4. 加载 emotion wheel 映射
5. 标签归一化和无效词过滤
6. 逐样本计算 precision 和 recall
7. 在 5 个 emotion wheels 上计算 F1
8. 对 5 个 F1 求平均得到最终分数
```

因此，想要提升最终成果，关键不是生成更长的解释，而是生成更准确、更规范、更适合 emotion wheel 映射体系的情绪标签。
